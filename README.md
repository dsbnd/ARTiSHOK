**АРТиШОК** — это платформа для цифровизации деятельности современных художественных галерей. Система решает проблему хаотичной организации выставок, предоставляя единую онлайн-среду для взаимодействия кураторов галерей и художников.

Организация выставки часто сопряжена с разрозненной коммуникацией (в мессенджерах и почте), сложностями планирования пространства и потерей информации о произведениях. 

Наш сервис устраняет эти проблемы:
* **Централизация данных:** Платформа служит надежным источником правды обо всех авторах, выставках и инвентаре галереи.
* **Упрощение логистики:** Визуальное планирование на интерактивной карте позволяет заранее продумать развеску картин и расстановку скульптур, ускоряя монтаж экспозиции.
* **Цифровой архив:** Система автоматически формирует историю всех прошедших мероприятий и участвовавших в них работ для аналитики и отчетности.

## Ключевой функционал

### 1. Интерактивная карта и визуальное планирование
Реализован модуль "Карта зала" (на базе `Leaflet.js`). Владельцы галерей могут загружать графические планы выставочных пространств (файлы хранятся в `MinIO`) и с помощью инструментов разметки выделять доступные стенды, указывая их тип (стена, подиум, открытое пространство) и размеры.

### 2. Умное бронирование 
Художники могут просматривать карту и бронировать конкретные места для своих работ. 
* **Валидация габаритов:** Система автоматически сравнивает длину, ширину и высоту произведения с максимально допустимыми размерами выбранного слота. При несоответствии система блокирует заявку.
* **Управление статусами:** Привязка строгих конечных автоматов к статусам стендов (Свободен/Забронирован) и заявок (Ожидает/Подтверждено/Отклонено) с помощью триггеров на уровне БД.

### 3. Защита данных и верификация
* **Ролевая модель доступа:** Изолированный функционал для Администраторов, Владельцев, Авторов и Анонимных посетителей.
* **Email-подтверждение:** Для защиты аккаунтов пользователей введно email-подтверждение при регистрации аккаунтов.

## Архитектура и Технологический стек

Проект построен по принципам многоуровневой клиент-серверной архитектуры, где Frontend и Backend общаются через RESTful API.

* **Backend:** Java, Spring Boot, Spring MVC.
* **ORM:** Hibernate.
* **Frontend:** JavaScript, React, Leaflet.js.
* **База данных:** PostgreSQL.
* **Инфраструктура:** `MinIO` (S3 хранилище для медиа), `MailHog` (email-тестирование), `Gradle` (сборка).

## Проектирование Базы Данных

БД спроектирована с упором на надежность и целостность данных.
* Включает более 10 сущностей с отношениями "многие-ко-многим" и каскадным удалением.
* Обеспечена строгая ссылочная целостность средствами DDL (проверки форматов email/телефонов через регулярные выражения, валидация хронологии дат выставок).
* Реализованы триггеры для автоматической валидации ролей и синхронизации статусов слотов при создании/отмене брони.
* Внедрены `B-Tree` индексы для оптимизации поиска свободных мест на конкретных картах и ускорения фильтрации списка заявок владельцем.
